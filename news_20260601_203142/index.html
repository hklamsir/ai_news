🚀 什麼是 Claude Code Routines？

Routines 本質上是「排程執行的 Claude Code 任務」。它啟動一個獨立的 Claude 會話，依照設定的排程（每小時、每天、或 GitHub webhook 觸發）反覆執行特定任務。與一般對話不同，Routines 在後臺自主運作，不需要人工在場。

每個 Routine 包含三個要素：排程觸發器（何時跑）、任務指令（做什麼）、許可權限制（能用什麼工具）。

「每天早上九點，Claude Code 自動檢查 GitHub Issues、分類優先順序、跑測試、甚至發起修復 PR。你起床時，程式碼已經被維護好了。」

— BlockTempo

🛠️ 三步建立第一個 Routine

第一步：安裝與確認環境

確認 Claude Code CLI 版本在 v2.0 以上：

claude --version

若版本不夠新，用官網安裝指令更新：

npm install -g @anthropic-ai/claude-code

第二步：建立 Routine 設定檔

在專案根目錄建立 .claude/routines/ 資料夾，然後建立設定檔 daily-tests.yml。

YAML 語法包含關鍵欄位：

  • name：任務名稱
  • schedule：cron 表示式
  • start_command：任務指令
  • allowed_tools：允許使用的工具清單
  • max_turns：最大對話輪數，防止無限迴圈

第三步：註冊與啟動

# 註冊 Routine
claude routines add --file .claude/routines/daily-tests.yml

# 啟動背景服務
claude routines start

# 檢視所有 Routine
claude routines list

# 檢視執行日誌
claude routines logs --name daily-tests

💼 三大實戰案例

🏷️

自動化 Issue 分類

每 30 分鐘掃描新 Issue，根據標題與標籤自動分類、回覆初步資訊，嚴重 bug 直接指派給對應開發者。設定 trigger.type: polling 定期輪詢 GitHub API。

🔍

每日程式碼健康檢查

每日凌晨執行，依序執行 lint 檢查、型別檢查、單元測試，若發現錯誤則自動建立分支修復並發起 PR。建議設定較高的 max_turns（50-100）。

📝

自動檔案更新

每日掃描新增的類別與函式，自動更新 README 和 API 檔案。這是最安全的 Routine 案例，因為檔案修改的風險遠低於程式碼修改。

⚡

Dynamic Workflow 互補

Dynamic Workflow 是人工即時引導的動態協作；Routines 是背景排程自動化。先用手動 Dynamic Workflow 測試任務流程，確定正確後再封裝成 Routine。

🔒 安全最佳實踐

⚠️ 四大安全鐵律

1. 永遠限制 allowed_tools，不要讓 Routine 擁有推送遠端分支的許可權

2. 設定 max_turns 上限，防止 Routine 無窮迴圈消耗 Token

3. 搭配 git 隔離策略：讓 Routine 在獨立分支上操作，經人工審查後合併

4. 從簡單任務開始（如 Issue 分類），確認行為正確後再升級到更複雜的任務
