GitHub Copilot 新收費制
開發者怒火中燒

從固定月費改為 Token 用量計費，用戶帳單暴漲數十倍

📺 資料來源：TechCrunch

📅 發佈日期：2026-05-30

🤖 處理方式：AI 智能摘要 + 深度優化

🎯 事件概覽

Microsoft 旗下的 GitHub Copilot 宣佈將於 2026 年 6 月 1 日起，由原本的固定月費訂閱制，改為按實際 token 使用量計費。新制上路後，開發者可以根據對話複雜度和請求次數，被收取遠高於過往的費用。

「什麼笑話。」一名 Reddit 用戶抱怨，原本每月只需約 $29 美元，新制卻可能使他每月帳單暴漲至近 $750 美元（約新臺幣 24,000 元）。

— Reddit r/GithubCopilot 網友

💸 價格爭議：用戶反映帳單飆漲

新制上路後，多名開發者在 Reddit 和 X（前 Twitter）上反映帳單金額大幅飆升：

  • $29 美元/月 → 暴漲至 $750 美元/月
  • $50 美元/月 → 飆升至 $3,000 美元/月

有用戶直言：「這個新收費模式貴得離譜，我已經取消訂閱了。這樣的價格，在實際應用上完全沒有成本效益。」

🔄 Microsoft 的商業邏輯

部分用戶指出，先前固定月費制下，Microsoft 補貼了大量「Vibe Coding」（純靠 AI 輔助開發、缺乏實際編程知識）的使用模式。有開發者表示：「這種瘋狂的用量，只有那些完全依賴 AI、不斷產生大量無用迭代的人才會遇到。」

另一名用戶在 Reddit 發問：「該死的，Copilot 到底虧了多少錢？」顯示出先前訂閱制定價的經濟邏輯確實令人困惑。

⚖️ 誰該負責？

批評新制的用戶認為，Microsoft 同時扮演了推波助瀾的角色——他們不斷簡化 AI 的使用門檻，變相鼓勵用戶無節制地消耗 token，現在又突然改變遊戲規則。

✨ 業界觀察

GitHub Copilot 的定價轉變，反映了 AI 輔助開發工具在商業化過程中的根本矛盾：如何平衡用戶期望、開發成本與可持續營收？對小型開發者而言，依賴 AI 編碼的代價正變得越來越高昂。

💰

價格飆漲

從固定 $29-$50 美元/月，飆升至 $750-$3,000 美元/月，讓小型開發者和獨立工作者難以負擔。

🎮

Vibe Coding 代價

批評者認為，暴增的用量主要來自「Vibe Coding」——不懂程式的人純靠 AI 輔助寫代碼的行為。

⚠️

Microsoft 的責任

支援者認為，Microsoft 鼓勵用戶無節制使用 AI，現在改變規則是「過河拆橋」。

📉

小型開發者受害

大型企業仍可負擔新制，但小型公司和獨立開發者正面臨成本失控的風險。
