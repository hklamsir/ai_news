💡 重點解析

Archon 的核心 primitive 是工作流文件——一個定義任務、輸入、輸出和步驟依賴關係的 YAML 文檔，具有可審查性和版本可控性。

  • 確定性輸出：透過 YAML 結構化定義，確保每次執行結果一致
  • 可組合性：工作流可跨項目複用，實現「定義一次，永久運行」
  • 版本控制：所有提示和工作流均可版本化管理
  • 多平台支援：CLI、GitHub Actions、Telegram、Slack 等無縫整合
  • 預包裝工作流：涵蓋從修復 GitHub Issue 到建立 PRD 等多種場景
