# OpenClaw 龍蝦之父教你省錢！開源工具「skill-cleaner」幫你的 AI 技能大減肥

## 📖 新聞導語

OpenClaw 開發者、被社群尊稱為「龍蝦之父」的 Peter Steinberger，無法忍受許多開發者將 AI 技能（Skill）描述寫得像書本一樣冗長，導致上下文窗口被無效雜訊塞滿，因而親自撰寫並開源了名為「skill-cleaner」的技能體檢工具。他呼籲開發者：**「別再塞垃圾進上下文了！」**

## 💡 核心內容

### ⚠️ 技能冗長推高運行成本與雜訊干擾

Peter 認為，AI 技能提示詞的冗長問題不只是文字描述的長短，而是**直接反映在每一次 API 呼叫的 Token 帳單上**。多寫一句廢話，每次調用就得支付額外的運行成本。

後果包括：
- **額外計費**：每個無效 Token 都增加 API 成本
- **決策混亂**：Agent 接收無效資訊越多，決策品質越差
- **延遲增加**：上下文越長，處理時間越長
- **注意力分散**：模型被無效內容干擾

###🧹 skill-cleaner 五大核心功能

這款開源工具具备五大核心功能：

| 功能 | 說明 |
|------|------|
| **技能提示詞預算審計** | 計算技能佔用的上下文令牌空間，分析預算比例，給出優化方案 |
| **重複技能檢測** | 跨越 Codex 內置庫、快取與個人根目錄，掃描並標記高度相似或同名的冗餘技能 |
| **未使用技能篩查** | 基於歷史日誌，找出長期未被呼叫的閒置技能以供清理 |
| **技能根目錄審計** | 統計所有技能來源並整理載入鏈路 |
| **描述精簡優化** | 主動篩選出冗長的技能描述，推薦簡化語法以節省預算 |

### 📝 極簡設計哲學

Peter 主張 AI 技能的設計應該要像**「路標」**一樣精簡：
- 路標的唯一任務是為 Agent 指明道路
- 不該把整本說明書掛在路標上

他自己開源的 `skill-cleaner` 就是最佳示範——核心技能檔案 `Skill.md` **僅有 56 行提示詞**。

他甚至在社群用極簡的「穴居人風格」（caveman style）示範省 Token：
> `install skill, agent smart, user happy`

堪稱將省錢刻入骨子里。

## 🔮 業界展望

隨著大型語言模型被應用在更複雜的工作流中，高雜訊的上下文不僅產生額外計費，還會讓 Agent 出現決策混亂。將冗餘文字精簡、將複雜邏輯封裝在底層程式碼中，才是打造高效、省錢且高可靠性 AI 系統的正確方向。
