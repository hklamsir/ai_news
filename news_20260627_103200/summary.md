# Anthropic產能暴增8倍有代價！Claude Code主管揭管理痛點，AI如何讓團隊陷入「孤島」危機？

## 📖 新聞導語

Anthropic 公布旗下工程師使用 Claude Code 後，每季交付程式碼量為 2021-2025 年平均值的 **8 倍**。但產能暴增同時，驗證工作量也同步暴增。Claude Code 與 Cowork 團隊工程負責人 Fiona Fung 在 Lenny's Podcast 訪談中揭示：寫程式不再是瓶頸，真正的挑戰是當程式碼供給速度遠超人類審核速度時，管理者該如何應對。

## 💡 核心內容

### 產能暴增背後的管理難題

根據 Anthropic 2026 年 6 月 4 日發布的《When AI builds itself》報告，工程師產能提升 8 倍，但 Fung 強調「程式碼行數偏重量而非品質」，高估實際生產力增幅。她觀察到關鍵問題：**程式碼供給量暴增的同時，驗證端工作量也同步暴增**。

### Fung 的管理原則：「Trust but Verify」

Fung 分享她的具體做法：所有 repo 開一個 Claude Code 遠端 session，賦予存取 Slack 頻道和內部指標權限。每天早上 Routines 自動掃描回饋頻道，整理使用者問題主題並產出 PR 草稿。她醒來就有摘要和 PR 等著她。

她也把 Claude Code 導入月度 1:1 管理會議——讓 Claude 即時分析這個月做了什麼、有沒有達到效果、用戶回饋有哪些主題。但她同時坦承：**當同時有 20 個代理在跑，context switching 的認知負荷其實在增加**，以前排出「進入心流的時間」，現在改成排出「消化非同步代理輸出的時間」。

### 招募策略：只鎖定兩種人

Fung 在招募上只鎖定：
1. **創造型工程師**（creative builders with product sense）
2. **深度系統專家**（deep systems experts，負責驗證）

她認為：介於兩者之間的一般化角色，已不是主要缺口。

### AI 讓人人高效，也讓人人變成「孤島」

Fung 被問到什麼事情讓她睡不著覺？她的答案是**文化維持**，而非技術挑戰。她的噩夢是：管理者說「都沒問題」，但背後其實已經很不妙——就像「管家在失火的房間裡安靜喝咖啡」的梗圖。

她形容：「文化是一個有生命、會呼吸的有機體，而不僅僅是掛在牆上的一張海報。」這套高效管理系統讓工程師幾乎不需要跨組溝通就能完成大量工作，這是效率來源，也是孤立感來源。

對抗孤立感的方法：Claude Code 團隊啟動「programming lunches」，讓工程師帶著各自的 Claude session 一起吃飯工作，互相觀察對方怎麼用工具——這不是技術活動，是**對抗孤立感的社交設計**。

### 菜鳥工程師怎麼教？

Fung 對新一代工程師的培養路徑感到不確定。以前工程師透過親手寫程式累積對架構和底層的直覺判斷，這個過程無法跳過。現在新工程師可以不讀程式碼就直接提交功能，這個「理解層」要怎麼建立？

她提出的可能方向是**師徒制**（fellowship 或 apprenticeship model）：讓新進工程師在資深人員陪伴下完成一定數量的實務工作。但她也說，這只是一個方向，不是答案。

## 🔮 業界展望

Fung 的觀察揭示 AI 輔助開發時代的深層矛盾：工具讓每個人都能達成更高成就，但「什麼都不能自動化」的判斷——文化維持、人與人的連結、對同事工作的觀察——反而成了最稀缺的能力。她的「programming lunches」實驗或許是個起點，但如何在高速增長中刻意放慢觀察同事，將是所有 AI 時代管理者必須回答的問題。
