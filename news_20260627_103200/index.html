Anthropic產能暴增8倍有代價！
Claude Code主管揭管理痛點

AI如何讓團隊陷入「孤島」危機？

📺 資料來源：BNEXT

📅 發布日期：2026-06-25

🤖 處理方式：AI 智能摘要生成

🎯 核心觀點

Claude Code 讓 Anthropic 工程師產能暴增 8 倍，但當 AI 能寫幾乎所有程式碼時，管理者最大的挑戰不再是技術，而是如何避免團隊陷入「孤島」危機，以及如何維持團隊文化。

📈

產能暴增 8 倍

根據 Anthropic 2026 年 6 月發布的《When AI builds itself》報告，旗下工程師平均每季交付的程式碼量約為 2021-2025 年平均值的 8 倍。但官方也提醒：程式碼行數偏重量而非品質，幾乎肯定高估實際生產力增幅。

⚠️

驗證工作量同步暴增

Claude Code 主管 Fiona Fung 的核心觀察：程式碼供給量暴增的同時，驗證端的工作量也同步暴增。當 AI 能寫所有程式碼，人類的角色變成審核者——而審核速度成為新瓶頸。

🔐

Trust but Verify

Fung 的管理原則是「trust but verify」：模型很強，但仍有需要深度人工判斷的區域，尤其是分散式系統等高複雜度場景。她在招募上只鎖定兩種人：創造型工程師（creative builders）與深度系統專家（deep systems experts）。

🏝️

高效但孤島化

這套高效管理系統讓工程師幾乎不需要跨組溝通就能完成大量工作——這是效率來源，也是孤立感來源。Fung 的噩夢：管理者說「都沒問題」，但背後其實已經很不妙。

🍽️

Programming Lunches 對抗孤立

Claude Code 團隊啟動「programming lunches」，讓工程師帶著各自的 Claude session 一起吃飯工作，互相觀察對方怎麼用工具。Fung 說：「文化是一個有生命、會呼吸的有機體，而不僅僅是掛在牆上的一張海報。」

📌 重點提示

當 AI 能寫所有程式碼，「什麼不能自動化」反而成了最稀缺的能力——文化維持、人與人的連結、對同事工作的觀察。

「寫程式已不再是瓶頸，它反而拉高了任何人所能達成成就的上限。」

— Fiona Fung，Claude Code 與 Cowork 團隊工程負責人

🔮 業界展望

Fung 的觀察揭示 AI 輔助開發時代的深層矛盾：工具讓每個人都能達成更高成就，但如何在高速增長中刻意放慢觀察同事、如何保存團隊的多元觀點和一體感，將是所有 AI 時代管理者必須回答的問題。她提出的師徒制（fellowship model）方向值得關注，但目前仍無標準答案。
