# Claude Code 都是誰在用？Anthropic 公開分析報告：非工程師暴增，專業知識反而更重要

## 📖 新聞導語

Anthropic 公布 2025 年 10 月至 2026 年 4 月的 Claude Code 使用數據，分析約 23.5 萬名使用者、近 40 萬個互動 session，發現非工程師用戶大幅增加，專業領域知識反而比過往更重要。

## 💡 核心內容

### 使用模式：修 Bug 變少，操作軟體變多
Claude Code 的工作分為九種模式，典型用途仍以開發為主：
- **修復壞掉的東西**：占 26%
- **建立新東西**：占 25%

但用途已明顯外擴：
- **操作軟體**：占 17%
- **撰寫文件與簡報**：占 10%
- **理解系統**：占 7%
- **規劃變更**：占 7%
- **代理與自動化 pipeline**：占 3%
- **資料分析**：占 3%

從時間變化來看，修 Bug 比例從 33% 降至 19%，操作軟體從 14% 升至 21%，寫作與資料分析合計從 10% 升至 20%。

### 任務價值上升 27%
Anthropic 用自由接案市場數據估算，發現平均任務價值在七個月內上升 27%：
- 建立新東西：+43%
- 操作軟體：+34%
- 修復類任務：+32%

### 用戶結構：非工程師快速成長
從可識別職業的 session 來看，最大宗仍是「電腦與數學相關職業」，但成長最快的非軟體職業是**管理、銷售和法律**相關工作。商業與金融業務、藝術設計、媒體、生命科學、社會科學等領域用戶亦顯著增加。

### 專家 vs 新手：專業知識決定產出效率
使用者在 session 中負責 70% 規劃決策，但僅 20% 執行決策，其餘交給 Claude 執行。

專業程度影響顯著：
- **新手**：每個 prompt 觸發約 5 個動作，輸出約 600 字，驗證成功率 15%
- **專家**：每個 prompt 觸發超過 12 個動作，輸出約 3,200 字，驗證成功率 28%–33%

最大差距在**新手與中階**之間，意味著只需對領域有一定理解，任務成功率就會大幅提升。

## 🔮 業界展望

Claude Code 已不再是工程師專屬工具，正變成各行各業的日常工作助手。關鍵在於：能用清晰語言描述需求、判斷結果品質、發現 AI 錯誤的人，將在 AI 時代最具競爭力。專業領域知識不是被 AI 取代，而是變得更有價值。
