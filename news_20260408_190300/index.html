Claude Code 原始碼外洩事件！6 大底層設計揭秘：為什麼它比其它 AI 好用？

新聞導語 就在上週，Anthropic 不小心把 Claude Code 的原始碼外洩了，一個年收估計 25 億美金的產品，1900 多個檔案、51 萬行程式碼全部被攤在陽光下。Anthropic 的 CEO Dario Amodei 自己說過，公司內部有 70%-90% 的程式碼是 Claude 寫的。Claude Code 的創始工程師 Boris Power 也表示自己已經好幾個月沒有親手改過一行 code。這次外洩事件揭示了 AI 時代的真相：懂得跟 AI 協作的團隊，效率可以是別人的好幾倍，但風險也同樣放大。

  • 原始碼外洩規模：1900 多個檔案，51 萬行程式碼，年收 25 億美金產品
  • AI 編程比例：Anthropic 內部 70%-90% 程式碼由 Claude 生成
  • 核心發現：AI 模型只佔系統 20%，80% 是底層基礎建設
  • 6 大底層設計：Permission 框架、Session 持久化、Budget 管理、系統日誌、系統瘦身、模組化
  • 安全機制：單一命令需通過 18 個獨立安全模組檢查
  • 深度防禦：分層安全檢查，任何一層漏掉下一層還有機會攔住

詳細分析

第一大設計：Permission 框架（權限管理）

  • Beware（最高信任）：內建核心功能，如讀檔案、搜尋程式碼，永遠可用
  • 中等信任：第三方開發的擴充功能，可以被關掉
  • Skill（最低信任）：使用者自定義能力，預設最低信任
  • 留德檢查：是否為安全指令（如 ls、git status）
  • 破壞性指令偵測：刪除資料、覆蓋檔案等不可復原操作
  • Git 安全檢查：防止誤刪 Production Branch
  • Sandbox 判定：決定是否需要在隔離環境執行

這種設計叫「深度防禦」（Defense in Depth），就像機場安檢有多道關卡，每一層負責不同類型的風險。

  • 207 個面向用戶的工具
  • 184 個面向 Model 的工具
  • 每個工具都要登記名字、來源、功能、描述
  • 根據任務模式只載入需要的工具，不是每次都載入全部

第二大設計：Session 持久化（狀態保存）

  • 對話內容
  • 每一輪用了多少 Token
  • 哪些操作已經被授權
  • 當前設定是什麼
  • 每個重要事件發生後都會存檔
  • Conversation State：我們聊到哪了
  • Workflow State：做到第幾步，哪些事情已經實際執行
Checkpoint 機制： 每個會對外部世界產生改變的步驟都要打 checkpoint，每個操作都有唯一編號，重啟後系統看到編號就知道已做過，不會重複執行。

第三大設計：Budget 管理（成本控制）

事前預估： 每一輪對話之前，先預估這一輪大概會用掉多少 Token，如果預估超過 Budget，直接在 API 呼叫發出之前擋掉。
  • 短期對 Anthropic 營收不利，但換取長期信任
  • 類似明碼標價，讓使用者知道該付多少費用
  • 快要用完時主動通知，進入收尾模式
  • 不會突然斷在半路讓使用者不知所措

第四大設計：系統日誌（System Logging）

兩層日誌系統：
  • 每個動作都即時廣播狀態
  • 選了哪個工具、決策是什麼、Token 用了多少
  • 使用者可以即時介入修正方向
  • 獨立於對話之外
  • 記錄所有系統層級事件
  • 每個事件都有分類和結構化細節
  • 用於事後 Debug
健康檢查（Doctor Pattern）： 專門指令檢查 API Credential 是否過期、外部連線、設定檔完整性、工具載入狀態。

第五大設計：系統瘦身（Optimization）

三個瘦身層面：
  • 根據 Mode、Permission、Context 動態組裝
  • 只載入本次對話需要的工具
  • 超過一定 turn 數自動 Compact
  • 是 Summarize 不是 Truncate
  • 保留 Context 同時減少 Token
  • 七個階段：帳號驗證 → 掃描 Project → 檢查環境 → 初始化工作區 → 載入 Plugin → 決定模式 → 進入主迴圈
  • 在用戶打第一個字之前已完成所有前置作業

第六大設計：模組化（Modularity）

  • Explore、Plan、Verification、Guide 等不同 Agent 類型
  • 每種有自己的 Prompt、Allow tools、Deny tools
  • 嚴格限制每個角色能做什麼
  • 8 個記憶模組
  • 每筆記憶打相關性分數
  • 分類：團隊共用、個人、Project 限定
  • Precision Memory（長期）vs Section Memory（當次對話）
  • 每筆記憶都要追溯來源（Provenance）
  • 20+ 個 Skill 模組
  • 每個 Skill 自帶觸發條件、Prompt 模板、工具需求
  • 獨立可插拔模組，擴充新能力只需寫 Skill，不用改核心架構

業界展望

  • Model 本身能力只佔 20%，真正的護城河在應用層框架
  • Harness（把 Model 包起來的系統）才是關鍵
  • 包含安全、穩定、效率、擴展性的所有設計
  • 讓 Model 能在真實世界可靠運作
  • Agent 教學通常只教那 20%（Model 使用）
  • 剩下 80% 的底層設計才是生產環境的關鍵
  • 不需要讀完 51 萬行 code，需要建立對 Harness 重要性的認知

結論

Claude Code 之所以好用，不是因為 Claude 的 Model 比別人好多少，而是因為那 80% 的底層設計：18 道安全模組、完整的 Session Persistence、Budget Check、七階段啟動流程、模組化的 Skill 系統。這些設計創造了那個「這工具太好用了」的 Aha Moment。
