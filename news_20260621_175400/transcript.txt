這時候，瑞幸在人類眼裡

這是一組可以被調用的服務能力
可以查附近門店，查商品，建立訂單，管理訂單
這會帶出一個全新的問題
也就是你的品牌要怎麼被找到？

所以我們做 App ，做網站，買廣告版位
請小編天天在社群上刷存在感

這才是最關鍵的轉變
未來的品牌不只要被人看見
還要能被 AI 調用
再往下看產品形態
過去十幾年，大部分 App 都是頁面流
進首頁，再進分類頁，再進商品頁
再加購物車，再結帳
產品經理就在這條路徑上做優化
按鈕大一點，優惠券露出早一點
流程短一點，新人優惠跳出來快一點
這套邏輯的核心，是人
因為是人需要看頁面，點按鈕，判斷選項
最後做決定
但 Agent 入口更像任務流
使用者不會手把手告訴 Agent 把每一頁都走一遍
只會說
幫我在公司附近買一杯我平常喝的咖啡
少冰無糖， 30 分鐘後我要拿到
這句話，對人來說只是交代需求
但對 Agent 來說，是一串工作
Agent 要先判斷你的意圖
你是要在 30 分鐘後拿到一杯符合偏好的咖啡
它要讀情境
現在幾點？你在哪裡？
公司附近門市叫什麼？你平常喝什麼？
它還要比較不同服務
路易莎，星巴克，超商
哪個離你最近？哪個有優惠？
然後它要整理方案，讓你確認
最後才是付款，下單，通知你去取餐
問題就在這裡
中間任何一步都可能出錯
商品缺貨，付款失敗，取餐時間來不及
這些在 App 裡本來是使用者自己一步一步看
一步一步確認的
但 Agent 接入之後
這些判斷都會變成 AI 要處理的任務
所以未來的入口競爭
可能不只是誰的 App 首頁更漂亮
而是誰的服務更容易被 Agent 接進來
誰的任務更容易被 Agent 跑完
講到這裡，要先補一個名詞，叫 AX
全名是 Agent Experience
這是 Netlify 的執行長 Mathias Biilmann
在 2025 年初提出的詞
他說 UX 是設計給人用的體驗
AX 就是設計給 AI Agent 用的體驗
UX 大家應該都聽過
就是 User Experience ，使用者體驗
以前 UX 關心的是
人看不看得懂頁面
知不知道下一步要點哪裡
會不會在結帳前放棄
過去產品設計幾乎都是圍繞 UX 展開的
但如果 AI Agent 會代表人使用服務
產品就多了一個新的使用者
這個使用者不是人，是 Agent 
而 AX ，也就是 Agent Experience
你可以把它理解成給 Agent 的使用體驗
AX 關心的不是頁面漂不漂亮
而是 Agent 能不能理解這個服務到底可以做什麼
並順利的達成目標
比如對一個咖啡服務來說
光是讓 Agent 知道這裡能點咖啡還不夠
每一個動作需要哪些資訊？
哪幾步一定要你本人點頭授權？
哪些情況絕對不能讓 AI 自己決定？
每一個能力需要哪些參數
哪些步驟需要人類授權
哪些情況不能自動執行
還要知道失敗的時候該怎麼辦
門店關了，要不要換門店？
商品沒了，能不能推薦替代品？
付款失敗，能不能告訴你失敗原因？
下錯單，能不能取消？
取消不了時，責任誰承擔？
這些以前不是 App 頁面最核心的設計問題
但在 Agent 入口裡
它們變得更重要了
所以未來產品要同時服務兩個使用者
真正做決定的人
和替人執行任務的 Agent
而這也會改寫平台現在的位置
因為今天很多生意
都是靠平台入口在運轉
外送平台，電商平台都是這樣
使用者先打開平台，再搜尋，比較，下單
平台掌握流量，也掌握排序和推薦
所以商家要配合平台規則
要買曝光，要做排名
要跟著平台的轉化邏輯走

他不說我要打開某某 App 
他只說 幫我買一杯咖啡
這時候平台的能力就不只是提供商品選項了
是提供穩定的交易，支付，履約
還有售後和風控
以前平台最值錢的是前台流量
誰先被使用者打開
誰就有機會分發注意力
未來平台的價值可能更多轉向後台能力
誰的下單更穩
誰的商品和庫存資料更清楚
誰的授權規則更安全
誰的售後流程更可追溯
誰就更容易被 Agent 選中
總的來說
是整體設計都要變得更加 AI-friendly 
這個變化很微妙
平台跟使用者的關係會被洗牌
過去平台掌握的是你看到什麼
未來它可能要競爭的是
Agent 願不願意調用你
這裡就要進入到下一個概念
AXO ， Agent Experience Optimization 
AX 是你的服務對 Agent 來說好不好用
AXO 就是把這件事給優化好
讓 Agent 在好幾個品牌裡比較時
更容易找到你，看懂你，然後選你
你可以直接對照 SEO 來理解
SEO 是讓搜尋引擎找到你，把你排前面
AXO 是讓 AI Agent 選到你
願意用你的服務把事情辦完
在 AXO 我們除了問有沒有這個功能以外
也會問當 AI 在多個品牌之間比較時
它憑什麼優先選擇我們？
這也是為什麼開 API 不是終點
很多人會以為
給 Agent 用的產品
就是把 API 開出去就好了
不是，你要設計的是一整條任務路徑
Agent 要能理解你的功能，知道邊界

在關鍵節點讓人確認
也要能把結果講回人聽得懂的語言
不然就算 API 接上了卻不好用
Agent 連到工具之後，也可能誤解任務
涉及付款，個資
錯誤成本就會變高
很多品牌可能只是想追一波 AI 熱點
並沒有真的重構流程

所以未來的入口可能不是某個 App
也不是某個聊天框
而是一個能把生活雜事整理成任務卡的 Agent 層
它會有幾個元素
第一是意圖入口
它不只要理解你想完成什麼
可能還要根據你的使用習慣預測，給出推薦
第二是情境理解
時間，地點，習慣，預算還有限制
這些通通都要被包含進來
第三是任務卡片
它不能只丟給你一段文字
而是要把建議變成可以確認，修改，拒絕的操作
第四是漸進授權
低風險的事情可以更自動
高風險的事情必須先問你
像是幫你繳水電費可以
但是如果這個月電費突然變成 2000 塊
你會希望它先跟你回報異常狀況讓你 double check 
第五是後台工具調用
因為它真的要能連到外送，電商平台，支付系統
不然就只是提醒，不是 Agent 
回到現在

今天 App 已經夠方便了
常購，再來一單，收藏

AI 再幫你省十幾秒

這些繁瑣的小事還有摩擦

是我沒有講清楚？
