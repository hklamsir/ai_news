# AI News 網頁生成 Prompt（新聞風格 - 精簡版）

請根據以下新聞素材，生成一個完整的**科技感新聞專題網頁**。

---

## 【重要要求】

### ❌ 禁止：
- **不要直接複製摘要內容**
- 不要使用 Markdown 符號（##、###、** 等）
- 不要使用「本影片」、「影片中」等字詞
- **Hero Section 圖片不要和文字重疊**

### ✅ 必須：
- **用你的理解重新組織內容**，分成清晰的段落
- 使用專業新聞用語（「本次報告」、「研究指出」、「數據顯示」）
- 將內容分成 3-5 個小節，每節有明確主題
- 使用 HTML 標籤（`<h3>`、`<p>`、`<ul>`、`<li>`）排版
- **Hero Section 圖片使用 grid 佈局，不要和文字重疊**
- **必須包含完整的 Footer**

---

## 【版面配置】

### 必備 Section（只包含這 3 個）：
1. **Hero Section** - 標題 + 短摘要 + 圖片（使用 grid/flex 佈局）
2. **Content Section** - 重新組織後的新聞內容
3. **Footer** - 版權信息 + 連結

---

## 【Hero Section 佈局要求】

**必須使用以下結構**（文字在上，圖片在下，使用 grid）：

```html
<section class="hero">
    <div class="hero-content">
        <span class="hero-badge">📰 AI News 專題報道</span>
        <h1>標題</h1>
        <p class="hero-subtitle">短摘要...</p>
        <div class="hero-images">
            <div class="hero-image-card">
                <img src="圖片路徑" alt="新聞插圖">
            </div>
        </div>
    </div>
</section>
```

### 1. Meta Info（資料來源）
```html
<div class="meta-info">
    <p><strong>📺 資料來源</strong>：<a href="https://youtu.be/_y772bb9VZE?si=nasBXDwvZMTidAY0" target="_blank">https://youtu.be/_y772bb9VZE?si=nasBXDwvZMTidAY0</a></p>

    <p><strong>🤖 處理方式</strong>：AI 智能摘要生成</p>
</div>
```

### 2. 內容分段（3-5 節，每節有主題）

根據素材內容，自動提煉 3-5 個小節主題，例如：

```html
<h3>🎯 核心觀點</h3>
<p>用 1-2 段話說明核心內容...</p>

<h3>💡 重點解析</h3>
<p>詳細說明重點內容...</p>
<ul>
    <li>要點一</li>
    <li>要點二</li>

<h3>🔮 未來展望</h3>
<p>總結並展望未來...</p>
```

### 3. 可選 Highlight Box（強調重要信息）
```html
<div class="highlight-box">
    <h4>✨ 關鍵洞察</h4>
    <p>重要內容...</p>
</div>
```

## 【內容風格】

### 新聞用語：
- ✅ 使用：「本次報告」、「研究指出」、「數據顯示」、「業界觀察」、「專家分析」
- ❌ 禁止：「本影片」、「影片中」、「這段影片」、「該影片」

### 排版要求：
- 每節 2-4 段，每段 3-5 句
- 使用 `<ul>` 列出要點（3-6 個）
- 重要概念用 `<strong>` 強調
- 不要使用 Markdown 符號

新聞導語

隨著生成式人工智慧技術快速滲透軟體開發領域，「Vibe Coding」——即不經深思熟慮、僅憑直覺讓AI生成程式碼的做法——正引發業界廣泛憂慮。研究指出，過度依賴AI自動生成程式碼已導致大量開發團隊面臨程式碼庫快速劣化、護成本激增的困境。針對此現象，TypeScript社群重量級人物Matt Pocock近期於GitHub公開其個人實踐的AI工程方法論，以「Skills for real engineers」為核心理念，強調軟體工程基礎在AI時代非但未過時，反而成為開發者最核心的競爭優勢。該倉庫上線後迅速累積逾8萬星標，引發全球開發社群熱烈迴響，被視為對抗「AI程式碼債務」現象的重要參考框架。

重點摘要

1. AI程式設計三大痛點浮現：需求理解偏差、冗長無效溝通，以及因生成速度過快導致的架構急速劣化，已成為開發者使用AI工具時的普遍困境。

2. 「GrealMe」技能重塑需求分析：透過反向盤問機制，迫使開發者在撰寫程式碼前將模糊概念具體化，顯著降低需求理解落差。

3. 「Greal with docs」建立領域知識庫：將團隊特定術語與決策沉澱為可複用的Context Markdown檔案，確保AI輸出與團隊規範高度一致。

4. 測試驅動開發融入AI工作流：以垂直切片方式執行紅綠重構迴圈，讓AI在受控框架內逐步生成程式碼，而非一次性大量輸出。

5. 定期架構審查對抗技術債務：「Improve codebase architecture」技能引導AI從全域視角檢視模組設計，主動識別複雜介面與潛在缺陷。

6. 人機協作新正規化確立：人類專注問題定義與架構設計，AI負責程式碼執行與重構，形成互補而非替代的協作關係。

詳細分析

資料顯示，Matt Pocock開源的Skills倉庫在GitHub平臺獲得超過8萬星標，反映開發社群對於結構化AI工程方法論的強烈需求。業界觀察指出，當前市場上諸如Cline、Devin、SpecKit等試圖「接管」完整開發流程的工具，雖標榜自動化效率，卻往往剝奪開發者的控制權，使除錯難度倍增。Pocock明確批判此類工具的路線，主張以小型、可組合的技能模組取代單一巨型系統，並確保與任何大型語言模型皆可相容。

專家分析認為，Pocock方法論的核心價值在於將經典軟工程實踐重新錨定於AI時代。以「GrealMe」技能為例，其名稱源自英文「grill」（盤問），設計理念顛覆傳統人機互動模式：AI不再扮演順從的程式碼生成器，而是轉型為嚴格的技術面試官。當開發者提出「建立登入頁面」等模糊需求時，AI會連續追問密碼重置流程、第三方登入支援、錯誤提示設計等關鍵細節，強迫決策者在編碼前完成深度思考。研究指出，此機制有效解決軟體開發中最常見的「對齊失敗」問題——即開發者誤以為AI理解需求，實則雙方認知存在根本落差。

在領域知識管理層面，「Greal with docs」技能針對AI「空降」專案時的上下文缺失問題提出解方。該技能要求AI在互動過程中持續檢視專案既有的領域語言與Context Markdown檔案，一旦偵測到模糊或衝突術語便即時追問澄清，並將確認後的關鍵決策沉澱至知識庫。資料顯示，這種做法不僅減少AI的無效冗言，更能確保變數命名、函式設計與團隊慣例保持一致，顯著提升跨成員協作效率。

面對AI生成速度過快導致的架構劣化風險，Pocock將測試驅動開發（TDD）與架構審查機制嵌入AI工作流。「TDD」技能嚴格遵循「紅綠重構」迴圈，要求AI逐個行為撰寫失敗測試、實現功能、執行重構，而非一次性輸出大量程式碼。這種垂直切片策略確保每個步驟皆可驗，使AI輸出維持在可控範圍。更進一步，「Improve codebase architecture」技能建議開發者定期執行全域審查，由AI從模組設計角度識別介面複雜、難以測試、易生缺陷的區域，相當於為程式碼庫建立定期健檢機制。

值得關注的是，Pocock的跨界背景——曾為聲樂教練後轉型頂尖工程師——使其對溝通精準度具備獨特敏銳度。這種經驗轉化為對「人機協作」本質的深刻洞察：AI作為強大執行者，缺乏對複雜系統的全域把控能力；若開發者自身不具備軟體工程素養，AI將以數倍速度製造技術債務。業界觀察強調，過去撰寫劣質程式碼或需數日，如今AI僅需數秒即可生成數千行問題程式碼，若不加以控制，專案崩解速度將遠超傳統開發模式。

業界展望

隨著開源大型語言模型能力持續提升，AI程式設計的技術門檻預期將進一步降低，但專家分析指出，未來開發者的核心競爭力將從「撰寫程式碼速度」轉向「駕馭AI管理複雜系統的能力」。頂尖工程師的定位正逐漸轉型為「AI架構師」——既精通底層技術，又擅長設計人機協作流程，知道何時放權、何時約束，在效率提升與品質管控間取得平衡。Pocock的實
